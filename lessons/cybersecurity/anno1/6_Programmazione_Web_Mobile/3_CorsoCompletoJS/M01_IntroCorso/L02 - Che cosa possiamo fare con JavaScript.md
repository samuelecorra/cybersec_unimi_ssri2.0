## **Lezione 2: Che cosa possiamo fare con JavaScript**

### **1. Dal “far comparire cose” al vero potenziale**

Nel video precedente abbiamo visto esempi base di JavaScript: far comparire elementi, gestire lo scroll, pulsanti “torna su”, piccoli pannelli nei Pokedex, editor di testo interattivi.  
Questi esempi servono a mostrare la **dinamicità** che JavaScript introduce sul web.

Ma questa è solo la superficie.  
In realtà JavaScript può fare **molto di più**, e il motivo è che intorno a lui esiste un **ecosistema gigantesco**.

Questa lezione serve proprio a capire **che cosa ci permette di fare**, dalla forma più “vanilla” fino agli strumenti avanzati.

### **2. Il vero focus del corso**

Una cosa va chiarita subito:  
imparare a programmare _in sé_ non è difficile. Lo puoi imparare:

- da me,
    
- da YouTube,
    
- da Udemy,
    
- dalla documentazione ufficiale.
    

Il valore di questo corso non è solo “imparare JavaScript”, ma:

**capire JavaScript moderno e capire il suo ecosistema.**

Come in CSS (dove abbiamo studiato anche il suo ecosistema di strumenti), qui imparerai come muoverti nella foresta di framework, librerie e strumenti che sono cresciuti sopra JavaScript negli ultimi dieci anni.

### **3. L’ecosistema di JavaScript**

Appena inizi a guardarti intorno trovi un’esplosione di nomi:

- **Node.js**
    
- **Express**
    
- **Angular**
    
- **React**
    
- **Vue**
    
- **Svelte**
    
- **Solid**
    
- **Preact**
    
- **Prettier**
    
- **ESLint**
    
- **Babel**
    
- **TypeScript**
    

Chi arriva oggi si trova davanti un universo enorme.  
Quando io ho iniziato, era molto più semplice: pochi strumenti, pochi framework, meno confusione.  
Dal 2012–2013 in poi è esploso tutto.

In CSS è successa la stessa cosa: CSS puro → Bootstrap → Bulma → Foundation → Sass → SCSS → Tailwind…  
Stessa identica dinamica anche in JavaScript.

### **4. JavaScript “Vanilla”: la base pura**

Come nel gaming, “vanilla” vuol dire _senza mod_.

**JavaScript Vanilla è JavaScript puro, senza framework, senza librerie.**

Tutto ciò che vedremo parte da questa base.  
Con JavaScript vanilla puoi costruire tu stesso:

- un editor di testo come quello visto prima,
    
- un Pokedex che filtra i dati,
    
- animazioni e pannelli dinamici.
    

È più complesso, ma si può fare.

Da qui nasce tutto il resto.

### **5. JavaScript può fare molto più che siti web**

Il suo ecosistema è talmente potente che puoi usarlo per creare:

- **siti web**
    
- **applicazioni web interattive**
    
- **applicazioni desktop**
    
- **app mobile iOS e Android**
    
- **backend e API**
    

JavaScript può essere **ovunque**.

#### **5.1 Applicazioni desktop – Electron.js**

Con **Electron** puoi creare applicazioni desktop complete.

Esempi reali costruiti con Electron (JavaScript):

- **Skype Desktop**
    
- **Visual Studio Code**
    
- **Figma**
    
- **Discord**
    
- **Dropbox**
    
- **Trello**
    
- **Postman**
    
- **WordPress Desktop**
    
- **GitHub Desktop**
    
- **Twitch**
    

Electron è potentissimo: unisce JavaScript + Node + un browser integrato.  
Tutto ciò che vedi su queste app è JavaScript che gira come software desktop.

#### **5.2 Applicazioni mobile – React Native**

React, creato da Facebook, ha generato un “fratello”:

**React Native**, per creare app mobile iOS e Android.

Le app di Facebook, Instagram, molte parti di WhatsApp e altri giganti sono state costruite con React e React Native.

Per il mobile esistono anche alternative come Flutter (che non è JavaScript), ma React Native resta uno dei protagonisti assoluti.  
È parte dell’ecosistema di JavaScript.

#### **5.3 Applicazioni web – React, Angular, Vue**

Framework come **React**, **Angular**, **Vue**, **Svelte**, **Solid**, **Preact** permettono di costruire siti web moderni, interattivi, complessi.

Facebook, Instagram, Netflix, New York Times, Dropbox, Vivaldi…  
Tutte queste interfacce sono scritte con framework JavaScript.

Il punto chiave: **interfacce web dinamiche su larga scala**.

### **6. Single Page Application (SPA)**

Le **SPA** sono applicazioni web dove la pagina **non si ricarica mai**.

Esempi:

- Netflix
    
- Pannelli aziendali e gestionali
    
- App che usi nel browser ma sembrano software vero
    

Su Netflix, ad esempio:

- clicchi e i contenuti cambiano,
    
- parte un video,
    
- si aprono pannelli,
    
- ma la pagina **non si ricarica mai**.
    

Questo è possibile solo grazie a JavaScript, che controlla ogni parte della UI e aggiorna solo ciò che serve, quando serve.

### **7. JavaScript lato server – Node.js**

JavaScript non è più solo nel browser.

Con **Node.js** possiamo:

- creare API,
    
- costruire backend,
    
- collegarci ai database,
    
- gestire utenti, login, pagamenti,
    
- generare file,
    
- eseguire script sul server,
    
- creare servizi web completi.
    

Altri linguaggi fanno lo stesso (Java Spring Boot, PHP Laravel, Python Django…), ma JavaScript è diventato uno degli standard grazie alla sua flessibilità.

Oggi puoi avere:

- **frontend in JavaScript**
    
- **backend in JavaScript**
    
- **mobile in JavaScript**
    
- **desktop in JavaScript**
    

È uno dei motivi per cui il linguaggio è esploso.

### **8. Siamo solo all’inizio**

In questi 10 minuti ti ho citato:

- framework,
    
- librerie,
    
- strumenti di build,
    
- concetti moderni,
    
- tecnologie ibride.
    

E siamo ancora **sulla punta dell’iceberg**.

Il vero obiettivo di questo corso è che, una volta finito, tu sappia:

1. **programmare in JavaScript moderno**, senza dubbi
    
2. **capire l’ecosistema**, riconoscendo a cosa serve ogni nuova tecnologia
    
3. **orientarti da solo nel mondo JavaScript**, che cambia continuamente
    

Devi arrivare al punto in cui, quando esce qualcosa di nuovo, dici:

> “Ok, è un framework per fare X. Lo metto nella categoria giusta.”  
> “Ok, questa è un’alternativa a Electron.”  
> “Ok, questo serve per il backend, come Node.”

Questa è la competenza vera.
