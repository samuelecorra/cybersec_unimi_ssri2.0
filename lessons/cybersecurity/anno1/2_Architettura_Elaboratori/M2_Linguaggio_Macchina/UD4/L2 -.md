# **Lezione 11: Sintesi Finale della LC-2 e Ponte verso il Linguaggio C (pagg. 81–87)**

## **1. Chiusura del ciclo di studio sulla LC-2**

Con questa sezione si chiude definitivamente lo studio:

- della **struttura interna della CPU**,
    
- del **linguaggio macchina**,
    
- del **linguaggio Assembly**,
    
- della **gestione della memoria**,
    
- dell’**input/output**,
    
- dei **salti, cicli e sottoprogrammi**.
    

Da qui in avanti, tutto ciò che studierai in **C** sarà una **trasposizione ad alto livello di questi stessi meccanismi**.

---

## **2. La LC-2 come “modello ideale” di calcolatore**

La LC-2 non è progettata per essere veloce o potente, ma per essere:

- **formalmente pulita**,
    
- **strutturalmente completa**,
    
- **didatticamente perfetta**.
    

Essa contiene **tutti i componenti essenziali di un vero processore moderno**:

- CPU con CU e ALU,
    
- registri generali,
    
- registri speciali,
    
- memoria,
    
- bus,
    
- periferiche,
    
- sistema di interrupt simulato tramite TRAP.
    

---

## **3. Parallelismo concettuale tra LC-2 e computer reali**

Ogni elemento studiato sulla LC-2 ha un perfetto equivalente nei PC reali:

- **GPR** → registri x86 / ARM
    
- **PC** → Instruction Pointer
    
- **CC** → Flag Register
    
- **JSR / RET** → CALL / RET
    
- **BR** → JUMP condizionati
    
- **TRAP** → System Call
    
- **LDR / STR** → Load / Store
    
- **.BLKW** → array in RAM
    
- **.STRINGZ** → stringhe C terminate da `\0`
    

Questo significa che:

> **chi capisce davvero la LC-2, capisce l’architettura di qualsiasi CPU moderna.**

---

## **4. Dal registro alla variabile**

In LC-2:

- i **registri** contengono direttamente i dati,
    
- ogni operazione lavora **esplicitamente su registri**.
    

In C:

```c
int x;
x = x + 1;
```

Dietro le quinte:

- `x` viene caricato in un registro,
    
- viene sommato 1,
    
- il risultato viene riscritto in memoria.
    

In Assembly LC-2 questo era:

```
LD R1, X
ADD R1, R1, #1
ST R1, X
```

---

## **5. Dal BR agli if e while**

Ogni costrutto del C nasce dalle istruzioni di salto:

|C|LC-2|
|---|---|
|`if`|`BRz`, `BRp`, `BRn`|
|`while`|`BR` + salto all’indietro|
|`for`|contatore + `BR`|
|`break`|`BR` in uscita|

Non esiste nessun “if magico” nella CPU:  
esistono solo **salti condizionati basati sui flag**.

---

## **6. Dal JSR alle funzioni**

In C:

```c
f();
```

In LC-2:

```
JSR F
...
RET
```

Meccanismo reale:

1. l’indirizzo di ritorno viene salvato in **R7**,
    
2. il PC salta alla funzione,
    
3. la funzione termina con `RET`,
    
4. l’esecuzione riprende dal punto corretto.
    

Questo è esattamente lo stesso principio usato oggi nelle CPU reali.

---

## **7. Dalla memoria agli array**

In LC-2 un array è solo:

- una **zona continua di memoria**,
    
- riservata con:
    

```
.BLKW n
```

In C:

```c
int v[10];
```

È la stessa cosa:

- 10 celle consecutive in RAM,
    
- accesso con **base + offset**.
    

---

## **8. Dalle stringhe LC-2 alle stringhe C**

In LC-2:

```
.STRINGZ "CIAO"
```

In memoria:

```
'C' 'I' 'A' 'O' 0
```

In C:

```c
char s[] = "CIAO";
```

Stesso identico principio:

> **stringa = sequenza di caratteri + terminatore `\0`**

---

## **9. Dall’I/O con TRAP allo stdio in C**

In C:

```c
scanf("%d", &x);
printf("%d", x);
```

In LC-2:

- `TRAP x20` → leggi carattere
    
- conversione ASCII → numero
    
- `TRAP x21` → stampa carattere
    
- `TRAP x22` → stampa stringa
    

Il C **non elimina la complessità**, la **nasconde**.

---

## **10. Dal puntatore LC-2 al puntatore in C**

In LC-2:

- un registro contiene un **indirizzo di memoria**,
    
- con `LDR` e `STR` si accede al contenuto.
    

In C:

```c
int *p;
*p = 5;
```

Dietro le quinte:

- `p` contiene un indirizzo,
    
- `*p` è l’accesso indiretto a quella cella.
    

È esattamente l’indirizzamento **indiretto** della LC-2.
