# **Lezione 2.1: Instruction Set Architecture (ISA) della CPU LC-2**

---

## **1. Linguaggio macchina della LC-2**

Il **linguaggio macchina della LC-2** è:

- **estremamente semplificato**,
    
- progettato per essere:
    
    - facile da decodificare,
        
    - facile da simulare,
        
    - ideale per lo studio.
        

Ogni istruzione:

- è lunga **16 bit**,
    
- occupa **una cella di memoria**,
    
- è composta da:
    
    - **opcode (4 bit)**,
        
    - **operandi (12 bit)**.
        

---

## **2. Istruzione ADD – Somma tra registri**

### **Formato generale**

```
ADD DR, SR1, SR2
```

- **DR** → Destination Register (3 bit)
    
- **SR1** → Source Register 1 (3 bit)
    
- **SR2** → Source Register 2 (3 bit)
    

L’istruzione:

> somma i due valori a 16 bit contenuti in SR1 e SR2  
> e memorizza il risultato in DR.

Tutti i registri sono **GPR**, quindi codificati su **3 bit**.

---

### **Bit di controllo (flag immediato)**

Dopo SR1 è presente un **bit speciale (flag)**:

- **0 → somma tra registri**
    
- **1 → somma con immediato**
    

Nel caso visto sopra siamo nel caso:

> **ADD tra due registri**

---

## **3. ADD con immediato a 5 bit**

Formato:

```
ADD DR, SR, imm5
```

Qui:

- **imm5** è una costante su **5 bit in complemento a 2**
    
- va **estesa a 16 bit** prima della somma
    

### **Estensione del segno**

Se imm5 è:

- **positivo (MSB = 0)** → si aggiungono **zeri**
    
- **negativo (MSB = 1)** → si aggiungono **uno**
    

Formalmente:

```
imm5 → sign-extension → 16 bit
```

---

### **Complemento a 2 – Richiamo**

Con $n$ bit il range è:

$$  
-2^{n-1} \leq x \leq 2^{n-1} - 1  
$$

L’opposto si ottiene con:

1. inversione dei bit
    
2. somma di 1
    

---

## **4. Istruzione AND – Prodotto logico**

### **Formato tra registri**

```
AND DR, SR1, SR2
```

Esegue:

> prodotto logico **bit-a-bit** tra SR1 e SR2  
> e salva il risultato in DR.

---

### **AND con immediato**

```
AND DR, SR, imm5
```

⚠️ **Differenza fondamentale con ADD:**

- l’immediato **NON viene esteso in base al segno**
    
- viene esteso **solo con zeri**
    

Quindi:

```
imm5 → zero-extension → 16 bit
```

---

## **5. Istruzione NOT – Negazione logica**

Formato:

```
NOT DR, SR
```

Effetto:

> ogni bit di SR viene **invertito**  
> e salvato in DR

È una **negazione bit-a-bit**.

---

## **6. Istruzione LEA – Load Effective Address**

Già introdotta concettualmente, ora la formalizziamo completamente.

Formato:

```
LEA DR, PCoffset9
```

- **DR** → registro destinazione
    
- **PCoffset9** → offset su 9 bit in complemento a 2
    

L’indirizzo viene costruito così:

$$  
\text{indirizzo} = PC_{\text{MSB}}\ \& \ PCoffset_9  
$$

⚠️ **LEA carica un INDIRIZZO, NON un valore.**

Serve per:

- costruire indirizzi a 16 bit,
    
- aggirare il limite dei 12 bit disponibili per gli operandi.
    

---

## **7. Istruzione LD – Load diretto da memoria**

Formato:

```
LD DR, PCoffset9
```

Funzionamento:

1. costruisce l’indirizzo con:  
    $$  
    PC_{\text{MSB}} \ \& \ PCoffset_9  
    $$
    
2. **legge il contenuto della cella**
    
3. salva il valore in **DR**
    

⛔ Diversa da LEA:

- **LEA → carica l’indirizzo**
    
- **LD → carica il contenuto**
    

---

## **8. Istruzione LDI – Load Indirect**

Formato:

```
LDI DR, PCoffset9
```

Esegue:

1. primo accesso → trova un **indirizzo**
    
2. secondo accesso → usa quell’indirizzo per leggere il **dato reale**
    
3. carica il dato in **DR**
    

➡️ Consente di **uscire dalla pagina corrente**, perché il secondo indirizzo è completo a 16 bit.

---

## **9. Istruzione LDR – Load con registro base**

Formato:

```
LDR DR, BaseR, index6
```

- **BaseR** → registro base
    
- **index6** → offset su 6 bit
    

Operazione:

$$  
\text{indirizzo} = \text{contenuto(BaseR)} + \text{sign-ext(index6)}  
$$

Poi:

- legge il valore dalla memoria
    
- lo carica in **DR**
    

✅ Vantaggio enorme:

> consente l’accesso a **qualsiasi zona della memoria con un solo accesso**.

---

## **10. Istruzione ST – Store diretto in memoria**

Formato duale della **LD**:

```
ST SR, PCoffset9
```

Funzionamento:

- calcola l’indirizzo con il PC
    
- **scrive in memoria il contenuto di SR**
    

---

## **11. Istruzione STI – Store Indirect**

Duale della **LDI**:

```
STI SR, PCoffset9
```

Funzionamento:

- primo accesso → trova un indirizzo
    
- secondo accesso → scrive il valore di **SR** nella cella puntata
    

---

## **12. Istruzione STR – Store con registro base**

Duale della **LDR**:

```
STR SR, BaseR, index6
```

Scrive:

$$  
\text{Mem[BaseR + index6]} \leftarrow \text{SR}  
$$

Anche qui:

✅ accesso diretto a **tutto lo spazio di indirizzamento con un solo accesso**

---

# **Istruzioni di Controllo della CPU LC-2**

---

## **1. Ruolo delle istruzioni di controllo**

Le **istruzioni di controllo** servono a:

> **modificare il flusso sequenziale di esecuzione del programma**.

Senza queste istruzioni, l’esecuzione sarebbe sempre e solo:

```
istruzione 1 → istruzione 2 → istruzione 3 → ...
```

Con le istruzioni di controllo invece è possibile:

- creare **salti condizionati**,
    
- realizzare **cicli**,
    
- implementare **if, while, for**,
    
- gestire **sottoprogrammi**,
    
- interagire con il **sistema operativo** tramite trap.
    

Tutte queste istruzioni **usano ancora il meccanismo del PCoffset** per costruire gli indirizzi.

---

## **2. Istruzione BR – Branch (Salto condizionato)**

L’istruzione **BR** realizza un **salto condizionato** basato sul contenuto del registro **Condition Codes (CC)**.

Ricordiamo che **CC è formato da tre bit**:

- **N** → risultato negativo
    
- **Z** → risultato nullo
    
- **P** → risultato positivo
    

---

### **2.1 Semantica generale della BR**

Forma concettuale:

```
if (condizione verificata) then
    PC = PC + 1 + PCoffset9
else
    PC = PC + 1
```

Il salto avviene solo se **almeno una delle condizioni specificate nell’istruzione coincide con lo stato attuale del registro CC**.

---

### **2.2 Formato dell’istruzione BR**

```
BR n z p PCoffset9
```

I tre bit **n, z, p** indicano **quali condizioni sono abilitate**.

---

## **3. Tipologie di BR (tutti i casi)**

### **3.1 Salto incondizionato**

```
BR x3001
```

Nessuna lettera dopo BR significa:

- **n = 1**
    
- **z = 1**
    
- **p = 1**
    

Quindi:

> il salto avviene **sempre**, qualunque sia CC.

Formalmente:

```
BR ≡ BRnzp
```

---

### **3.2 Salto se positivo**

```
BRp x3001
```

- **p = 1**
    
- **n = 0**
    
- **z = 0**
    

Il salto avviene **solo se l’ultimo risultato è positivo**:

```
CC = 001
```

---

### **3.3 Salto se zero**

```
BRz x3001
```

- **z = 1**
    
- **n = 0**
    
- **p = 0**
    

Il salto avviene **solo se il risultato è nullo**:

```
CC = 010
```

---

### **3.4 Salto se negativo**

```
BRn x3001
```

- **n = 1**
    
- **z = 0**
    
- **p = 0**
    

Il salto avviene **solo se il risultato è negativo**:

```
CC = 100
```

---

### **3.5 Salto se zero O positivo**

```
BRzp x3004
```

- **z = 1**
    
- **p = 1**
    
- **n = 0**
    

Il salto avviene se:

```
CC = 010  oppure  CC = 001
```

---

### **3.6 Salto sempre (scrittura esplicita)**

```
BRnzp x3009
```

È **identico al BR senza lettere**:

> salto incondizionato esplicito.

---

## **4. Costruzione dell’indirizzo nel BR**

Come sempre l’indirizzo viene costruito così:

$$  
PC_{\text{nuovo}} = PC + 1 + \text{sign-ext}(PCoffset9)  
$$

L’offset è:

- su **9 bit**,
    
- in **complemento a 2**,
    
- quindi può rappresentare:
    

$$  
-256 \leq \text{offset} \leq 255  
$$

---

## **5. Istruzione JSR – Jump to SubRoutine**

La **JSR** serve a:

> chiamare un **sottoprogramma**.

Cosa fa realmente la JSR:

1. **salva l’indirizzo dell’istruzione successiva in R7**
    
2. **salta all’indirizzo del sottoprogramma**
    

---

### **5.1 Registro R7: Link Register**

Il registro **R7** viene usato come:

> **registro di ritorno**

Serve per sapere **dove tornare dopo la fine della subroutine**.

---

### **5.2 Costruzione dell’indirizzo in JSR**

Anche qui si usa:

- **PC**
    
- **PCoffset11**
    

L’indirizzo viene costruito con:

$$  
PC_{\text{MSB}} \ \& \ PCoffset_{11}  
$$

---

## **6. Istruzione JMP – Jump Register**

La **JMP** realizza un salto:

> usando **direttamente un registro come sorgente dell’indirizzo**.

Formato:

```
JMP BaseR
```

Effetto:

```
PC ← BaseR
```

È l’istruzione che consente:

- il **ritorno da sottoprogramma**,
    
- tramite:
    

```
JMP R7
```

---

## **7. Istruzione RET**

Formalmente **RET non è una vera istruzione** distinta:

```
RET ≡ JMP R7
```

Serve a:

> tornare all’istruzione successiva a quella che aveva chiamato la JSR.

---

## **8. Istruzione TRAP – Chiamata al sistema**

La **TRAP** consente alla CPU di:

> passare temporaneamente il controllo al **sistema operativo**.

Formato:

```
TRAP vect8
```

Dove:

- **vect8** è un codice su **8 bit**
    
- individua una **routine di sistema predefinita**
    

---

### **8.1 Funzionamento interno della TRAP**

Quando viene eseguita una TRAP:

1. l’indirizzo di ritorno viene salvato in **R7**
    
2. si accede alla **tabella delle TRAP**
    
3. si esegue la routine di sistema corrispondente
    
4. alla fine si torna indietro con **RET**
    

---

### **8.2 Esempi tipici di TRAP**

Le TRAP permettono di:

- stampare caratteri,
    
- leggere input da tastiera,
    
- stampare stringhe,
    
- terminare il programma.
    

Sono l’unico vero **ponte tra la CPU e il sistema operativo simulato**.

---

# ✅ **Le 4 TRAP principali della LC-2**

Le **TRAP** sono **routine di sistema già pronte**, fornite dal simulatore della LC-2, che permettono di fare **input/output e terminazione del programma** senza dover scrivere tutto da zero in Assembly.

Tu devi solo:

- mettere il valore giusto in **R0**
    
- chiamare la TRAP corretta
    

---

## ✅ **1) TRAP x21 — OUTPUT DI UN CARATTERE**

### 🔹 Cosa fa

Stampa **un solo carattere** sullo schermo.

### 🔹 Come funziona internamente

1. Legge il contenuto del registro **R0**
    
2. Lo interpreta come **codice ASCII**
    
3. Stampa il carattere corrispondente a video
    

### 🔹 Schema operativo

```
R0 = codice ASCII
TRAP x21
```

### 🔹 Esempio

Se:

```
R0 = x0041
```

x0041 = 65 decimale = ASCII di `'A'`

Allora:

```
TRAP x21  → stampa: A
```

### 🔹 Equivalente in C

```c
putchar(R0);
```

---

## ✅ **2) TRAP x22 — OUTPUT DI UNA STRINGA**

### 🔹 Cosa fa

Stampa **una stringa completa di caratteri**, a partire da un indirizzo in memoria.

### 🔹 Come funziona

1. Legge da **R0 un indirizzo di memoria**
    
2. Da quell’indirizzo:
    
    - legge un carattere
        
    - lo stampa
        
    - passa alla cella successiva
        
3. Continua **finché non trova lo 0 (terminatore NULL)**
    

### 🔹 Schema operativo

```
R0 = indirizzo della stringa
TRAP x22
```

### 🔹 Esempio

Se:

```
R0 = x3100
Mem[x3100] = 'H'
Mem[x3101] = 'i'
Mem[x3102] = 0
```

Allora:

```
TRAP x22  → stampa: Hi
```

### 🔹 Equivalente in C

```c
puts(R0);
```

(con R0 visto come puntatore a stringa)

---

## ✅ **3) TRAP x23 — INPUT DI UN CARATTERE**

### 🔹 Cosa fa

Legge **un carattere dalla tastiera**.

### 🔹 Come funziona

1. Il programma **si ferma in attesa**
    
2. L’utente preme un tasto
    
3. Il codice **ASCII del carattere digitato viene salvato in R0**
    

### 🔹 Schema operativo

```
TRAP x23
→ R0 = ASCII del carattere digitato
```

### 🔹 Esempio

Se l’utente preme:

```
B
```

ASCII di `'B'` = 66 = x0042

Allora:

```
R0 = x0042
```

### 🔹 Equivalente in C

```c
getchar();
```

---

## ✅ **4) TRAP x25 — TERMINAZIONE DEL PROGRAMMA (HALT)**

### 🔹 Cosa fa

**Ferma definitivamente il programma.**

### 🔹 Effetto reale

- Blocca l’esecuzione
    
- Arresta il flusso delle istruzioni
    
- Non ritorna al programma
    
- Chiude la simulazione
    

### 🔹 Schema operativo

```
TRAP x25
```

Oppure più semplicemente:

```
HALT
```

(perché `HALT = TRAP x25`)

### 🔹 Equivalente in C

```c
exit(0);
```

Va sempre **alla fine del `main`**.

---

# ✅ RIASSUNTO SECCO DA MEMORIZZARE

|TRAP|Funzione|Usa R0 come|Effetto|
|---|---|---|---|
|x21|Output carattere|codice ASCII|stampa 1 carattere|
|x22|Output stringa|indirizzo stringa|stampa fino a 0|
|x23|Input carattere|ritorno ASCII|legge un tasto|
|x25|HALT|—|termina il programma|

---


## **9. Significato didattico delle istruzioni di controllo**

Con **BR, JSR, JMP e TRAP** la LC-2 diventa:

- **programmabile realmente**,
    
- capace di gestire:
    
    - scelte,
        
    - cicli,
        
    - funzioni,
        
    - I/O.
        

Queste istruzioni sono:

> **il cuore logico della programmazione sulla LC-2**.
