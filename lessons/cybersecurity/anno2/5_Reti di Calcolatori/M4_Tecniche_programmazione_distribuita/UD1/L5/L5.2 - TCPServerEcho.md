`TCPServerEcho` implementa un semplice "Echo Server". Un programma server "Echo" rimane in ascolto su una specifica porta e attende richieste di connessione da parte di un Echo Client. Il server riceve il messaggio dal client e lo rinvia subito al client mittente.

## **Soluzione commentata**

> 📌 Lo schema del server è sempre: **crea socket → bind → listen → loop(accept → gestisci client)**. È fondamentalmente diverso dal client perché deve essere il server a stare in ascolto e aspettare che i client si connettano.

---

### **Step 1 — Funzione `HandleTCPClient(int clntSocket)`**

```c
void HandleTCPClient(int clntSocket)
```

**`int clntSocket`**: il file descriptor del socket *specifico* per quel client, restituito da `accept()`. Il server usa un socket distinto per ogni connessione: `servSock` serve solo per accettare nuove connessioni, `clntSocket` serve per comunicare con quel preciso client.

```c
if ((recvMsgSize = recv(clntSocket, echoBuffer, RCVBUFSIZE, 0)) < 0)
    printf("recv() failed");
```

**Prima `recv()`**: blocca finché non arrivano dati. Restituisce il numero di byte ricevuti. Se restituisce 0 il client ha chiuso la connessione; se < 0 c'è un errore.

```c
while (recvMsgSize > 0)
{
    if (send(clntSocket, echoBuffer, recvMsgSize, 0) != recvMsgSize)
        printf("send() failed");

    if ((recvMsgSize = recv(clntSocket, echoBuffer, RCVBUFSIZE, 0)) < 0)
        printf("recv() failed");
}
```

**Loop `while (recvMsgSize > 0)`**: continua finché il client invia dati. La condizione di uscita è `recvMsgSize == 0`, che indica che il client ha chiuso la connessione (TCP FIN ricevuto). Il loop fa:
1. **`send()`**: rispedisce esattamente `recvMsgSize` byte al client. Usa `recvMsgSize` (non `RCVBUFSIZE`) perché l'ultima `recv()` potrebbe aver letto meno di 32 byte.
2. **`recv()`**: attende altri dati dal client per il prossimo ciclo.

```c
close(clntSocket);
```

Chiude il socket del *client specifico* (non il socket server). Il server è pronto ad accettare la prossima connessione.

---

### **Step 2 — Dichiarazione variabili in `main()`**

```c
int servSock;                      // ① socket di ascolto del server
int clntSock;                      // ② socket della singola connessione con il client
struct sockaddr_in echoServAddr;   // ③ indirizzo locale del server
struct sockaddr_in echoClntAddr;   // ④ indirizzo del client che si è connesso
unsigned short echoServPort;       // ⑤ porta su cui il server ascolta
unsigned int clntLen;              // ⑥ dimensione della struttura echoClntAddr
```

**① `servSock` vs ② `clntSock`**: questa è la differenza fondamentale tra client e server. Il server ha sempre almeno due socket:
- `servSock`: il "campanello" — sta sempre in ascolto sulla porta configurata, non trasporta mai dati applicativi.
- `clntSock`: il "canale di conversazione" — creato da `accept()` per ogni client che si connette, trasporta i dati reali.

**⑥ `unsigned int clntLen`**: `accept()` vuole un puntatore alla dimensione della struttura indirizzo del client. Prima della chiamata va impostato a `sizeof(echoClntAddr)`; dopo la chiamata `accept()` lo aggiorna con la dimensione reale dell'indirizzo del client.

---

### **Step 3 — Creazione del socket server**

```c
echoServPort = atoi(argv[1]);

if ((servSock = socket(AF_INET, SOCK_STREAM, 0)) < 0)
    printf("socket() failed");
```

Identico al client: `AF_INET` per IPv4, `SOCK_STREAM` per TCP. Il server crea il socket *prima* di configurare l'indirizzo.

---

### **Step 4 — Bind: associare il socket alla porta**

```c
memset(&echoServAddr, 0, sizeof(echoServAddr));
echoServAddr.sin_family      = AF_INET;
echoServAddr.sin_addr.s_addr = htonl(INADDR_ANY);
echoServAddr.sin_port        = htons(echoServPort);

if (bind(servSock, (struct sockaddr *) &echoServAddr, sizeof(echoServAddr)) < 0)
    printf("bind() failed");
```

**`bind()`** è l'operazione che dice al kernel: "questo socket deve ricevere i pacchetti che arrivano su questa porta". Il client non chiama `bind()` perché il kernel assegna automaticamente una porta effimera; il server invece *deve* scegliere una porta nota su cui i client potranno connettersi.

**`INADDR_ANY`**: costante speciale che significa "accetta connessioni su qualsiasi interfaccia di rete locale" (loopback, Ethernet, Wi-Fi, ecc.). Se il server avesse più schede di rete e si volesse ascoltare solo su una, si metterebbe l'IP specifico di quella scheda.

**`htonl(INADDR_ANY)`**: anche `INADDR_ANY` (valore 0) va convertito in network byte order con `htonl()` (32 bit). In questo caso specifico è inutile (0 è 0 in qualsiasi endianness), ma è buona pratica sempre farlo.

> 💡 L'analogia: il client chiama un numero di telefono che conosce (la porta del server). Il server "registra" il suo numero presso la centrale (il kernel via `bind()`) e aspetta chiamate in entrata.

---

### **Step 5 — Listen: mettere il socket in ascolto**

```c
if (listen(servSock, MAXPENDING) < 0)
    printf("listen() failed");
```

**`listen()`** trasforma il socket da "generico" a "socket di ascolto passivo". I due argomenti:
1. `servSock`: il socket da mettere in ascolto.
2. `MAXPENDING = 5`: dimensione della **coda di backlog** — quante richieste di connessione TCP il kernel può tenere in attesa prima che il server chiami `accept()`. Se la coda è piena, le nuove richieste vengono scartate dal kernel.

> ⚠️ `listen()` non blocca il programma. Si limita a configurare il socket. Il blocco vero avviene alla chiamata successiva, `accept()`.

---

### **Step 6 — Loop principale: accept → gestisci client**

```c
for (;;)   // loop infinito: il server non si ferma mai
{
    clntLen = sizeof(echoClntAddr);

    if ((clntSock = accept(servSock, (struct sockaddr *) &echoClntAddr, &clntLen)) < 0)
        printf("accept() failed");

    printf("Handling client %s\n", inet_ntoa(echoClntAddr.sin_addr));
    HandleTCPClient(clntSock);
}
```

**`clntLen = sizeof(echoClntAddr)`**: deve essere impostato *dentro* il loop, prima di ogni `accept()`. `accept()` lo modifica in output, quindi al prossimo ciclo va re-inizializzato.

**`accept(servSock, (struct sockaddr *) &echoClntAddr, &clntLen)`**: la chiamata bloccante che attende una connessione. Quando un client esegue `connect()`:
- Il kernel completa il three-way handshake.
- `accept()` sblocca e restituisce un nuovo file descriptor (`clntSock`) specifico per quella connessione.
- `echoClntAddr` viene popolata con l'indirizzo IP e la porta del client che si è connesso.

**`inet_ntoa(echoClntAddr.sin_addr)`**: converte l'indirizzo IP del client dal formato binario (32 bit) alla stringa dotted-decimal leggibile (es. `"192.168.1.100"`). Usato solo per il log a schermo.

**`HandleTCPClient(clntSock)`**: dopo `accept()` passa il controllo alla funzione di gestione. In questo server *sequenziale* (non multithread), il server è bloccato in `HandleTCPClient()` finché il client non chiude la connessione: **gestisce un solo client alla volta**.

## **Codice soluzione completo**

```c
#include <stdio.h>
#include <sys/socket.h>
#include <arpa/inet.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>

#define MAXPENDING 5

void HandleTCPClient(int clntSocket)
{
    char echoBuffer[32];
    int recvMsgSize;

    if ((recvMsgSize = recv(clntSocket, echoBuffer, 32, 0)) < 0)
        printf("recv() failed");

    while (recvMsgSize > 0)
    {
        if (send(clntSocket, echoBuffer, recvMsgSize, 0) != recvMsgSize)
            printf("send() failed");

        if ((recvMsgSize = recv(clntSocket, echoBuffer, 32, 0)) < 0)
            printf("recv() failed");
    }
    close(clntSocket);
}

int main(int argc, char *argv[])
{
    int servSock;
    int clntSock;
    struct sockaddr_in echoServAddr;
    struct sockaddr_in echoClntAddr;
    unsigned short echoServPort;
    unsigned int clntLen;

    if (argc != 2)
    {
        fprintf(stderr, "Usage: %s <Server Port>\n", argv[0]);
        exit(1);
    }

    echoServPort = atoi(argv[1]);

    if ((servSock = socket(AF_INET, SOCK_STREAM, 0)) < 0)
        printf("socket() failed");

    memset(&echoServAddr, 0, sizeof(echoServAddr));
    echoServAddr.sin_family      = AF_INET;
    echoServAddr.sin_addr.s_addr = htonl(INADDR_ANY);
    echoServAddr.sin_port        = htons(echoServPort);

    if (bind(servSock, (struct sockaddr *) &echoServAddr, sizeof(echoServAddr)) < 0)
        printf("bind() failed");

    if (listen(servSock, MAXPENDING) < 0)
        printf("listen() failed");

    for (;;)
    {
        clntLen = sizeof(echoClntAddr);

        if ((clntSock = accept(servSock, (struct sockaddr *) &echoClntAddr, &clntLen)) < 0)
            printf("accept() failed");

        printf("Handling client %s\n", inet_ntoa(echoClntAddr.sin_addr));
        HandleTCPClient(clntSock);
    }
}
```

**Sintassi di lancio:**
```
TCPEchoServer <Server Port>
```

## **Confronto client vs server**

| Fase | Client | Server |
|---|---|---|
| Crea socket | `socket()` | `socket()` |
| Associa indirizzo | — (automatico) | `bind()` |
| Mette in ascolto | — | `listen()` |
| Connessione | `connect()` → avvia | `accept()` → attende |
| Invio dati | `send()` | `send()` (rispedisce) |
| Ricezione dati | `recv()` | `recv()` |
| Chiusura | `close(sock)` | `close(clntSock)` |

> ✅ Il server usa **due socket**: `servSock` per accettare connessioni (mai chiuso nel loop), `clntSock` per comunicare con il singolo client (chiuso da `HandleTCPClient` al termine della comunicazione).
