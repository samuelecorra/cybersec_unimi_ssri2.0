## **Soluzione commentata**

> 📌 Lo schema del client è sempre lo stesso: **crea socket → riempie struttura indirizzo → connect → send → recv loop → close**.

### **Step 1 — Dichiarazione delle variabili**

```c
int sock;                          // ① file descriptor del socket (intero come tutti i fd Unix)
struct sockaddr_in echoServAddr;   // ② struttura IPv4 per l'indirizzo del server
unsigned short echoServPort;       // ③ porta su 16 bit (0–65535), senza segno
char *servIP;
char *echoString;
char echoBuffer[RCVBUFSIZE];       // ④ buffer di caratteri per la risposta
```

**① `int sock`** — `socket()` restituisce un intero non negativo che funge da "nome" del canale di comunicazione, esattamente come `open()` restituisce un intero per i file. Se vale -1, la creazione è fallita.

**② `struct sockaddr_in echoServAddr`** — struttura IPv4 con tre campi utili: `sin_family` (famiglia AF_INET), `sin_addr` (IP binario a 32 bit), `sin_port` (porta in network byte order). È la forma specializzata di `sockaddr` per IPv4.

**③ `unsigned short echoServPort`** — le porte TCP/UDP sono numeri a 16 bit (0–65535). Si usa `unsigned short` perché non esistono porte negative e il valore massimo è 65535, che entra perfettamente in 16 bit senza segno.

**④ `char echoBuffer[RCVBUFSIZE]`** — array di caratteri in cui verranno scritti i byte ricevuti da `recv()`. La dimensione è 32: limitata apposta per dimostrare che la ricezione può avvenire in più chunk.

---

### **Step 2 — Gestione della porta**

```c
if (argc == 4) {
    echoServPort = atoi(argv[3]); // porta specificata dall'utente
} else {
    echoServPort = 7;             // porta Echo well-known (RFC 862)
}
```

**`argc == 4`**: il programma si chiama con `TCPEchoClient <IP> <Stringa> [<Porta>]`. Se ci sono 4 argomenti (incluso `argv[0]` = nome programma), allora l'utente ha specificato la porta. Altrimenti si usa il default.

**Porta 7 — Echo Protocol**: la porta 7 è la well-known port del servizio Echo definita nell'RFC 862. Un Echo Server standard ascolta esattamente su questa porta. Le porte < 1024 richiedono privilegi di root per essere aperte sul lato server.

**`atoi(argv[3])`**: converte la stringa dell'argomento a linea di comando (es. `"8080"`) in un intero. Non effettua validazione: se l'utente passa una stringa non numerica, restituisce 0.

---

### **Step 3 — Creazione del socket**

```c
if ((sock = socket(AF_INET, SOCK_STREAM, 0)) < 0)
    printf("socket() failed");
```

I tre argomenti di `socket()`:

| Argomento | Valore | Significato |
|---|---|---|
| `domain` | `AF_INET` | Address Family Internet — usa IPv4 |
| `type` | `SOCK_STREAM` | Stream socket — usa TCP (affidabile, orientato alla connessione) |
| `protocol` | `0` | Seleziona automaticamente il protocollo standard per il tipo scelto (TCP per SOCK_STREAM) |

Se si fosse voluto UDP si sarebbe usato `SOCK_DGRAM` al posto di `SOCK_STREAM`.

Il valore restituito è il **file descriptor** del socket: un intero ≥ 0 se tutto va bene, -1 in caso di errore. Il controllo `< 0` intercetta qualsiasi errore di creazione.

---

### **Step 4 — Popolamento della struttura indirizzo**

```c
memset(&echoServAddr, 0, sizeof(echoServAddr)); // azzera tutti i byte della struttura
echoServAddr.sin_family       = AF_INET;                    // IPv4
echoServAddr.sin_addr.s_addr  = inet_addr(servIP);          // IP server in binario
echoServAddr.sin_port         = htons(echoServPort);        // porta in network byte order
```

**`memset(..., 0, ...)`**: obbligatorio prima di usare la struttura. Azzera il campo `sin_zero[8]` (padding) e previene comportamenti indefiniti dovuti a memoria sporca.

**`AF_INET`**: specifica che si sta usando IPv4. Per IPv6 si userebbe `AF_INET6` con la struttura `sockaddr_in6`.

**`inet_addr(servIP)`**: converte la stringa IP in formato dotted-decimal (es. `"192.168.1.1"`) nel corrispondente valore binario a 32 bit **già in network byte order** (big-endian). Non serve `htonl()` dopo `inet_addr()` perché la funzione fa già la conversione internamente.

**`htons(echoServPort)`**: converte la porta dal formato dell'host (che potrebbe essere little-endian, come su x86) al network byte order (big-endian). Su macchine già big-endian la funzione non fa nulla, ma il codice resta portabile.

> ⚠️ Non usare mai la porta e l'indirizzo direttamente senza le funzioni di conversione: su architetture little-endian (praticamente tutti i PC moderni) i byte sarebbero invertiti e la connessione fallirebbe o raggiungerebbe l'host sbagliato.

---

### **Step 5 — Connessione al server**

```c
if (connect(sock, (struct sockaddr *) &echoServAddr, sizeof(echoServAddr)) < 0)
    printf("connect() failed");
```

`connect()` esegue il **three-way handshake TCP** (SYN → SYN-ACK → ACK) e blocca il programma finché la connessione non è stabilita o fallisce.

**Cast `(struct sockaddr *)`**: necessario perché l'API `connect()` accetta il tipo generico `sockaddr*` per essere indipendente dalla famiglia di indirizzi. Noi passiamo una `sockaddr_in` (IPv4), ma dobbiamo fare il cast esplicito al tipo generico.

**`sizeof(echoServAddr)`**: indica a `connect()` la dimensione effettiva della struttura passata, così sa quanti byte leggere.

> 💡 Dopo `connect()` con successo, la quaterna `(IP_locale, porta_locale, IP_server, porta_server)` è completamente definita. Il kernel assegna automaticamente una porta locale libera ≥ 1024 al client (porta effimera).

---

### **Step 6 — Invio del messaggio**

```c
echoStringLen = strlen(echoString);

if (send(sock, echoString, echoStringLen, 0) != echoStringLen)
    printf("send() sent a different number of bytes than expected");
```

`send()` invia i dati sul socket connesso. I quattro argomenti:
1. `sock` — il file descriptor del socket
2. `echoString` — il buffer con i dati da inviare
3. `echoStringLen` — quanti byte inviare
4. `0` — flags: nessun comportamento speciale

Il controllo `!= echoStringLen` è fondamentale: `send()` può restituire un numero di byte inferiore a quelli richiesti (partial send), specialmente se il buffer del kernel è pieno. In questo esempio didattico si segnala solo l'errore; in produzione si userebbe un loop per inviare il rimanente.

---

### **Step 7 — Ricezione della risposta (loop)**

```c
totalBytesRcvd = 0;
printf("Received: ");

while (totalBytesRcvd < echoStringLen)
{
    if ((bytesRcvd = recv(sock, echoBuffer, RCVBUFSIZE - 1, 0)) <= 0)
        printf("recv() failed or connection closed prematurely");

    totalBytesRcvd       += bytesRcvd;
    echoBuffer[bytesRcvd]  = '\0';
    printf(echoBuffer);
}
printf("\n");
```

**Perché il loop?** TCP è un protocollo a flusso di byte (*stream*): non garantisce che `recv()` riceva tutti i dati in una sola chiamata. Se la stringa è più lunga del buffer (32 byte), arriverà in più pezzi. Il loop continua finché `totalBytesRcvd` non raggiunge `echoStringLen`.

**`recv(sock, echoBuffer, RCVBUFSIZE - 1, 0)`**: il terzo argomento è `RCVBUFSIZE - 1` (31) e non `RCVBUFSIZE` (32) perché l'ultimo byte serve per il terminatore di stringa `'\0'` che aggiungiamo subito dopo.

**`echoBuffer[bytesRcvd] = '\0'`**: `recv()` non termina automaticamente la stringa. Senza questo, `printf(echoBuffer)` andrebbe a leggere memoria arbitraria oltre i dati ricevuti.

**`totalBytesRcvd += bytesRcvd`**: accumula il conteggio totale. Il loop termina solo quando si è ricevuto esattamente lo stesso numero di byte inviati.

---

### **Step 8 — Chiusura del socket**

```c
close(sock);
exit(0);
```

**`close(sock)`**: libera il file descriptor e avvia il processo di chiusura TCP (FIN → FIN-ACK → ACK). Senza `close()`, il socket resterebbe aperto fino alla fine del processo e le risorse del kernel non verrebbero liberate.

## **Codice soluzione completo**

```c
#include <stdio.h>
#include <sys/socket.h>
#include <arpa/inet.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>

#define RCVBUFSIZE 32

int main(int argc, char *argv[])
{
    int sock;
    struct sockaddr_in echoServAddr;
    unsigned short echoServPort;
    char *servIP;
    char *echoString;
    char echoBuffer[RCVBUFSIZE];
    unsigned int echoStringLen;
    int bytesRcvd, totalBytesRcvd;

    if ((argc < 3) || (argc > 4))
    {
        fprintf(stderr, "Usage: %s <Server IP> <Echo Word> [<Echo Port>]\n", argv[0]);
        exit(1);
    }

    servIP     = argv[1];
    echoString = argv[2];

    if (argc == 4) {
        echoServPort = atoi(argv[3]);
    } else {
        echoServPort = 7;
    }

    if ((sock = socket(AF_INET, SOCK_STREAM, 0)) < 0)
        printf("socket() failed");

    memset(&echoServAddr, 0, sizeof(echoServAddr));
    echoServAddr.sin_family      = AF_INET;
    echoServAddr.sin_addr.s_addr = inet_addr(servIP);
    echoServAddr.sin_port        = htons(echoServPort);

    if (connect(sock, (struct sockaddr *) &echoServAddr, sizeof(echoServAddr)) < 0)
        printf("connect() failed");

    echoStringLen = strlen(echoString);

    if (send(sock, echoString, echoStringLen, 0) != echoStringLen)
        printf("send() sent a different number of bytes than expected");

    totalBytesRcvd = 0;
    printf("Received: ");

    while (totalBytesRcvd < echoStringLen)
    {
        if ((bytesRcvd = recv(sock, echoBuffer, RCVBUFSIZE - 1, 0)) <= 0)
            printf("recv() failed or connection closed prematurely");

        totalBytesRcvd       += bytesRcvd;
        echoBuffer[bytesRcvd]  = '\0';
        printf(echoBuffer);
    }

    printf("\n");
    close(sock);
    exit(0);
}
```

**Sintassi di lancio:**
```
TCPEchoClient <Server IP> <Echo Word> [<Echo Port>]
```

- `Server IP`: indirizzo IP del server (`127.0.0.1` se entrambi girano sulla stessa macchina).
- `Echo Word`: stringa da inviare.
- `Echo Port`: (opzionale) porta su cui comunicare. Default: 7.

> ⚠️ Il client necessita sempre che il server sia già in esecuzione per poter funzionare.

## **Schema del flusso client**

```
main()
  │
  ├─ Parsing argomenti (IP, stringa, porta)
  │
  ├─ socket(AF_INET, SOCK_STREAM, 0)        → ottieni file descriptor
  │
  ├─ Popola sockaddr_in (IP + porta)
  │
  ├─ connect()                              → three-way handshake TCP
  │
  ├─ send()                                 → invia la stringa
  │
  └─ while (totalBytesRcvd < echoStringLen)
       └─ recv() → stampa chunk → aggiorna contatore
  │
  └─ close()                                → chiudi connessione TCP
```
