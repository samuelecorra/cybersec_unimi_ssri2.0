## **Consegna**

`TCPServerEchoSelect` implementa un server Echo in grado di ascoltare su **più porte contemporaneamente** utilizzando la funzione `select()` vista nella Lezione 6. Grazie a `select()`, si crea un insieme di socket su cui si può rimanere contemporaneamente in ascolto in lettura, in scrittura e in caso di eccezione. La `select()` sospende il programma fino allo scadere del timeout o fino a che non succede qualcosa sui socket in ascolto.

Il programma è diviso in tre funzioni oltre al `main`:
- **`AcceptTCPConnection`**: accetta la comunicazione in attesa sul socket `servSock` specificato.
- **`CreateTCPServerSocket`**: crea un singolo socket sulla porta `port` specificata.
- **`HandleTCPClient`**: gestisce la comunicazione con un singolo client su un socket specifico.

## **Soluzione commentata**

> 📌 La novità rispetto a L5.2 è `select()`: invece di bloccarsi su un solo `accept()`, il server usa `select()` per monitorare **tutti i socket contemporaneamente** e risponde solo a quelli pronti. Questo è il fondamento della programmazione di rete non bloccante a processo singolo.

---

### **Funzione 1 — `AcceptTCPConnection(int servSock)`**

```c
int AcceptTCPConnection(int servSock)
{
    int clntSock;
    struct sockaddr_in echoClntAddr;
    unsigned int clntLen;

    clntLen = sizeof(echoClntAddr);

    if ((clntSock = accept(servSock, (struct sockaddr *) &echoClntAddr, &clntLen)) < 0)
        printf("accept() failed");

    printf("Handling client %s\n", inet_ntoa(echoClntAddr.sin_addr));
    return clntSock;
}
```

**Perché separare `accept()` in una funzione?** Perché il `main` usa `select()` per determinare *quale* socket ha una connessione in attesa, poi chiama `AcceptTCPConnection()` passando *quel* socket specifico. La funzione è un wrapper pulito che nasconde i dettagli di `accept()`.

**`int clntSock`** — il valore restituito dalla funzione: il file descriptor del socket client, che verrà passato a `HandleTCPClient()`.

**`unsigned int clntLen`** — inizializzato a `sizeof(echoClntAddr)` *prima* di `accept()`. Dopo `accept()`, il kernel aggiorna questo valore con la dimensione effettiva dell'indirizzo del client. Per IPv4 sarà sempre `sizeof(struct sockaddr_in)`.

**`inet_ntoa(echoClntAddr.sin_addr)`** — converte l'IP binario del client nella stringa leggibile per il log. Nota: `inet_ntoa()` usa un buffer statico interno, quindi non è thread-safe.

---

### **Funzione 2 — `CreateTCPServerSocket(unsigned short port)`**

```c
int CreateTCPServerSocket(unsigned short port)
{
    int sock;
    struct sockaddr_in echoServAddr;

    if ((sock = socket(AF_INET, SOCK_STREAM, 0)) < 0)
        printf("socket() failed");

    memset(&echoServAddr, 0, sizeof(echoServAddr));
    echoServAddr.sin_family      = AF_INET;
    echoServAddr.sin_addr.s_addr = htonl(INADDR_ANY);
    echoServAddr.sin_port        = htons(port);

    if (bind(sock, (struct sockaddr *) &echoServAddr, sizeof(echoServAddr)) < 0)
        printf("bind() failed");

    if (listen(sock, MAXPENDING) < 0)
        printf("listen() failed");

    return sock;
}
```

Questa funzione incapsula le tre operazioni necessarie per creare un socket server pronto all'ascolto: `socket()` + `bind()` + `listen()`. Viene chiamata una volta per ogni porta specificata a linea di comando.

**Perché è una funzione separata?** Perché il `main` deve creare *N* socket (uno per ciascuna porta). Senza questa funzione, il codice nel `main` si ripeterebbe N volte. Separandola, si chiama `CreateTCPServerSocket(portNo)` in un loop: codice pulito e manutenibile.

**Il parametro `port`**: passato direttamente a `htons(port)`, è la porta locale su cui questo specifico socket ascolterà. Ogni chiamata produce un socket su una porta diversa.

**`return sock`**: restituisce il file descriptor del socket creato, che il `main` salva nell'array `servSock[]`.

### **Funzione 3 — `HandleTCPClient(int clntSocket)`**

```c
void HandleTCPClient(int clntSocket)
{

        if ((recvMsgSize = recv(clntSocket, echoBuffer, 32, 0)) < 0)
            printf("recv() failed");

    close(clntSocket);

```

Identica alla funzione vista in L5.2. Logica: `recv()` → loop(`send()` + `recv()`) finché il client chiude la connessione → `close()`. Vedere L5.2 per la spiegazione dettagliata.

### **Funzione `main` — la parte nuova: `select()`**

#### **Fase 1 — Parsing argomenti e allocazione array**

```c
timeout  = atol(argv[1]);
noPorts  = argc - 2;

servSock = (int *) malloc(noPorts * sizeof(int));
maxDescriptor = -1;
```

**`argc - 2`**: il programma si chiama con `TCPEchoServerSelect <timeout> <porta1> <porta2> ...`. `argv[0]` è il nome del programma, `argv[1]` è il timeout. Tutte le successive sono porte: quindi `argc - 2` è il numero di porte.

**`malloc(noPorts * sizeof(int))`**: alloca un array dinamico di interi. Non si può usare un array statico perché il numero di porte non è noto a compile-time. Ogni elemento è un file descriptor `int` (4 byte su sistemi a 32/64 bit).

**`maxDescriptor = -1`**: `select()` ha bisogno di sapere qual è il file descriptor più alto da controllare (argomento `nfds = maxDescriptor + 1`). Si parte da -1 e si aggiorna via via che si creano i socket.

---

#### **Fase 2 — Creazione dei socket server**

```c
for (port = 0; port < noPorts; port++)
{
    portNo         = atoi(argv[port + 2]);
    servSock[port] = CreateTCPServerSocket(portNo);

    if (servSock[port] > maxDescriptor)
        maxDescriptor = servSock[port];
}
```

Per ogni porta specificata a linea di comando:
1. Recupera il numero di porta da `argv[port + 2]` (partendo dall'indice 2 perché 0=programma, 1=timeout).
2. Crea il socket con `CreateTCPServerSocket()` e salva il file descriptor in `servSock[port]`.
3. Aggiorna `maxDescriptor` se il nuovo fd è più alto: `select()` controlla tutti i fd da 0 a `maxDescriptor` incluso.

> 💡 I file descriptor Unix vengono assegnati in ordine crescente a partire dal minimo disponibile (tipicamente 3, dato che 0/1/2 sono stdin/stdout/stderr). Quindi il `maxDescriptor` cresce con ogni socket creato.

---

#### **Fase 3 — Il loop principale con `select()`**

```c
while (1)
{
    // ① Reset dell'insieme
    FD_ZERO(&sockSet);

    // ② Aggiunge tutti i socket all'insieme
    for (port = 0; port < noPorts; port++)
        FD_SET(servSock[port], &sockSet);

    // ③ Imposta il timeout
    selTimeout.tv_sec  = timeout;
    selTimeout.tv_usec = 0;

    // ④ select(): blocca finché almeno un socket è pronto o scade il timeout
    if (select(maxDescriptor + 1, &sockSet, NULL, NULL, &selTimeout) == 0)
        printf("No echo requests for %ld secs...Server still alive\n", timeout);
    else
    {
        // ⑤ Controlla quale socket ha ricevuto una richiesta
        for (port = 0; port < noPorts; port++)
            if (FD_ISSET(servSock[port], &sockSet))
            {
                printf("Request on port %d: ", port);
                HandleTCPClient(AcceptTCPConnection(servSock[port]));
            }
    }
}
```

**① `FD_ZERO(&sockSet)`**: **obbligatorio prima di ogni chiamata a `select()`**. `select()` *modifica* l'`fd_set` in output (lascia impostati solo i bit dei socket pronti). Senza `FD_ZERO`, alla seconda iterazione l'insieme conterrebbe i risultati della `select()` precedente, non la lista completa dei socket da monitorare.

**② `FD_SET(servSock[port], &sockSet)`**: aggiunge il file descriptor `servSock[port]` all'insieme. Internamente imposta il bit corrispondente nel vettore di bit della struttura `fd_set`.

**③ `selTimeout`**: struttura `timeval` con `tv_sec` (secondi) e `tv_usec` (microsecondi). Impostata al valore `timeout` specificato dall'utente. Va re-impostata prima di ogni `select()` perché alcune implementazioni Unix la modificano in output (Linux la aggiorna con il tempo rimanente).

**④ `select(maxDescriptor + 1, &sockSet, NULL, NULL, &selTimeout)`**:
- Primo argomento: `maxDescriptor + 1` — il kernel controlla i fd da 0 a `maxDescriptor` incluso.
- Secondo argomento (`&sockSet`): `readfds` — i socket da monitorare in *lettura* (nuove connessioni in arrivo).
- Terzo argomento (`NULL`): `writefds` — nessun socket da monitorare in scrittura.
- Quarto argomento (`NULL`): `exceptfds` — nessun socket da monitorare per eccezioni.
- Quinto argomento (`&selTimeout`): timeout configurato dall'utente.
- **Ritorna 0**: timeout scaduto, nessun socket pronto.
- **Ritorna > 0**: N socket sono pronti (hanno una connessione in attesa).
- **Ritorna -1**: errore.

> ⚠️ I socket di tipo `SOCK_STREAM` in ascolto diventano "pronti in lettura" quando c'è una connessione in attesa nella coda (cioè quando `accept()` può essere chiamata senza bloccare). Non significa che ci siano dati da leggere, ma che una *nuova connessione* è pronta.

**⑤ `FD_ISSET(servSock[port], &sockSet)`**: dopo `select()`, l'`fd_set` contiene solo i socket pronti. `FD_ISSET` controlla se il bit del socket specifico è ancora impostato: se sì, quel socket ha una connessione in attesa.

**`HandleTCPClient(AcceptTCPConnection(servSock[port]))`**: composizione di funzioni:
1. `AcceptTCPConnection(servSock[port])` → chiama `accept()` su quel socket specifico → restituisce `clntSock`.
2. `HandleTCPClient(clntSock)` → gestisce la comunicazione echo → chiude `clntSock`.

---

#### **Fase 4 — Cleanup**

```c
for (port = 0; port < noPorts; port++)
    close(servSock[port]);

free(servSock);
exit(0);
```

Chiude tutti i socket server e libera la memoria allocata con `malloc()`. In questo programma il `while(1)` non termina mai, quindi questo codice non viene mai raggiunto; in una versione con condizione di uscita sarebbe essenziale per evitare memory leak e file descriptor leak.

## **Codice soluzione completo**

```c
#include <stdlib.h>

#include <unistd.h>

{

    struct sockaddr_in echoClntAddr;

    clntLen = sizeof(echoClntAddr);

    if ((clntSock = accept(servSock, (struct sockaddr *) &echoClntAddr, &clntLen)) < 0)
        printf("accept() failed");

    printf("Handling client %s\n", inet_ntoa(echoClntAddr.sin_addr));
    return clntSock;
}

int CreateTCPServerSocket(unsigned short port)
{
    int sock;
    struct sockaddr_in echoServAddr;

    if ((sock = socket(AF_INET, SOCK_STREAM, 0)) < 0)
        printf("socket() failed");

    memset(&echoServAddr, 0, sizeof(echoServAddr));
    echoServAddr.sin_family      = AF_INET;
    echoServAddr.sin_addr.s_addr = htonl(INADDR_ANY);
    echoServAddr.sin_port        = htons(port);

    if (bind(sock, (struct sockaddr *) &echoServAddr, sizeof(echoServAddr)) < 0)
        printf("bind() failed");

    if (listen(sock, MAXPENDING) < 0)
        printf("listen() failed");

    return sock;
}

void HandleTCPClient(int clntSocket)
{
    char echoBuffer[32];
    int recvMsgSize;

    if ((recvMsgSize = recv(clntSocket, echoBuffer, 32, 0)) < 0)
        printf("recv() failed");

    while (recvMsgSize > 0)
    {
        if (send(clntSocket, echoBuffer, recvMsgSize, 0) != recvMsgSize)
            printf("send() failed");

        if ((recvMsgSize = recv(clntSocket, echoBuffer, 32, 0)) < 0)
            printf("recv() failed");
    }
    close(clntSocket);
}

int main(int argc, char *argv[])
{
    int *servSock;
    int maxDescriptor;
    fd_set sockSet;
    long timeout;
    struct timeval selTimeout;
    int noPorts;
    int port;
    unsigned short portNo;

    if (argc < 3)
    {
        fprintf(stderr, "Usage: %s <Timeout (secs.)> <Port 1> ...\n", argv[0]);
        exit(1);
    }

    timeout  = atol(argv[1]);
    noPorts  = argc - 2;

    servSock = (int *) malloc(noPorts * sizeof(int));
    maxDescriptor = -1;

    for (port = 0; port < noPorts; port++)
    {
        portNo         = atoi(argv[port + 2]);
        servSock[port] = CreateTCPServerSocket(portNo);

        if (servSock[port] > maxDescriptor)
            maxDescriptor = servSock[port];
    }

    printf("Starting server: \n");
    while (1)
    {
        FD_ZERO(&sockSet);

        for (port = 0; port < noPorts; port++)
            FD_SET(servSock[port], &sockSet);

        selTimeout.tv_sec  = timeout;
        selTimeout.tv_usec = 0;

        if (select(maxDescriptor + 1, &sockSet, NULL, NULL, &selTimeout) == 0)
            printf("No echo requests for %ld secs...Server still alive\n", timeout);
        else
        {
            for (port = 0; port < noPorts; port++)
                if (FD_ISSET(servSock[port], &sockSet))
                {
                    printf("Request on port %d: ", port);
                    HandleTCPClient(AcceptTCPConnection(servSock[port]));
                }
        }
    }

    for (port = 0; port < noPorts; port++)
        close(servSock[port]);

    free(servSock);
    exit(0);
}
```

**Sintassi di lancio:**
```
TCPEchoServerSelect <Timeout (secs.)> <Port 1> <Port 2> ...
```

Esempio: `./TCPEchoServerSelect 10 7777 8888 9999` — server echo in ascolto su tre porte con timeout di 10 secondi.

## **Confronto L5.2 vs L6.1: select vs server sequenziale**

| Caratteristica | L5.2 (sequenziale) | L6.1 (con `select`) |
|---|---|---|
| N. porte in ascolto | 1 | N (parametro da riga di comando) |
| Meccanismo attesa | `accept()` bloccante | `select()` su N socket |
| Timeout configurabile | No | Sì |
| Client simultanei | No (uno alla volta) | No (uno alla volta, ma su N porte) |
| Comportamento senza richieste | Blocca per sempre | Stampa "still alive" ogni timeout |
| Complessità | Semplice | Moderata |

> ✅ `select()` non risolve il problema della gestione simultanea di *N client attivi* (per quello servono thread), ma risolve il problema dell'ascolto su *N porte diverse* senza bloccarsi su una sola.
