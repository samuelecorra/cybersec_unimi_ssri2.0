# L1 — Introduzione a SQL per SQL Injection

> 📌 SQL è il linguaggio usato per interrogare e gestire basi di dati relazionali. Per capire gli attacchi di SQL injection bisogna prima capire come una query seleziona colonne, legge tabelle, applica condizioni e può anche modificare o distruggere dati.

---

### **1. Obiettivo della Lezione**

In questa lezione viene introdotto brevemente il linguaggio **SQL**, in modo da poter trattare nella lezione successiva gli attacchi di tipo **SQL injection**.

L'obiettivo non è studiare SQL in modo completo.

Il corso di basi di dati tratterà questi argomenti in modo molto più approfondito.

Qui serve capire:
- che cos'è una base di dati relazionale;
- che cos'è una tabella;
- che cosa sono record e attributi;
- come si scrive una query SQL di base;
- che ruolo hanno `SELECT`, `FROM` e `WHERE`;
- che cosa sono proiezione e restrizione;
- come si combinano più tabelle;
- perché SQL non è solo un linguaggio di interrogazione;
- perché comandi come `DROP` e `INSERT` sono rilevanti per la sicurezza.

> 💡 La SQL injection sfrutta il fatto che molte applicazioni web costruiscono query SQL usando input ricevuto dall'utente. Se l'input viene inserito nella query senza controllo, può diventare parte del comando SQL eseguito.

---

### **2. Basi di Dati Relazionali**

Una base di dati relazionale è un insieme di tabelle.

Ogni tabella rappresenta una collezione di informazioni omogenee.

Per esempio, in un'applicazione aziendale si possono avere tabelle per:
- clienti;
- ordini;
- agenti di vendita;
- prodotti;
- pagamenti.

#### **2.1. Righe come Record**

Le righe di una tabella rappresentano **record**.

Un record descrive un singolo oggetto o evento.

Esempi:
- un cliente;
- un ordine;
- un prodotto;
- un agente di vendita.

#### **2.2. Colonne come Attributi**

Le colonne rappresentano gli **attributi** dell'oggetto.

Per esempio, per un cliente gli attributi possono essere:
- cognome;
- nome;
- città;
- sconto.

Per un ordine gli attributi possono essere:
- codice cliente;
- codice agente;
- prodotto venduto;
- data;
- costo.

> 📌 Una tabella relazionale organizza oggetti in righe e proprietà degli oggetti in colonne.

---

### **3. Query SQL**

Una **query** è un'interrogazione rivolta alla base di dati.

La query specifica quali informazioni vogliamo ottenere.

Il risultato di una query è ancora una tabella.

Questa tabella:
- può contenere molte righe;
- può contenere una sola riga;
- può essere vuota;
- può contenere solo alcune colonne della tabella originale;
- può combinare dati provenienti da più tabelle.

In SQL, la forma base di una query di interrogazione è:

```sql
SELECT attributi
FROM tabelle
WHERE condizioni;
```

La clausola `WHERE` è opzionale.

---

### **4. Clausola SELECT**

La clausola `SELECT` specifica quali attributi vogliamo vedere nel risultato.

In altre parole, determina le colonne della tabella in output.

Esempio:

```sql
SELECT cognome, nome, citta
FROM clienti;
```

Questa query chiede al database di restituire solo le colonne:
- `cognome`;
- `nome`;
- `citta`.

Non restituisce tutte le informazioni dei clienti, ma solo gli attributi richiesti.

#### **4.1. Asterisco**

Se si vuole restituire tutte le colonne, si usa:

```sql
SELECT *
FROM clienti;
```

L'asterisco significa:

> restituisci tutti gli attributi della tabella.

> 📌 `SELECT` controlla quali colonne compaiono nel risultato.

---

### **5. Clausola FROM**

La clausola `FROM` specifica da quali tabelle leggere i dati.

Esempio:

```sql
SELECT *
FROM clienti;
```

In questo caso la query legge dalla tabella `clienti`.

Se la query deve usare più tabelle, la clausola `FROM` può contenere più nomi:

```sql
SELECT clienti.nome, ordini.prodotto
FROM clienti, ordini
WHERE clienti.codiceCliente = ordini.codiceCliente;
```

Questa forma permette di combinare informazioni provenienti da tabelle diverse.

> 📌 `FROM` controlla quali tabelle vengono usate per costruire il risultato.

---

### **6. Clausola WHERE**

La clausola `WHERE` specifica condizioni che le righe devono soddisfare.

È opzionale.

Se manca, la query non applica restrizioni sulle righe.

Esempio:

```sql
SELECT *
FROM ordini
WHERE costo > 100;
```

Questa query restituisce solo gli ordini con costo maggiore di 100.

#### **6.1. Condizioni Booleane**

La condizione in `WHERE` è un'espressione booleana.

Può essere costruita combinando predicati semplici.

Gli operatori di confronto includono:
- minore;
- maggiore;
- uguale;
- diverso;
- minore o uguale;
- maggiore o uguale.

In SQL si possono usare operatori logici come:
- `AND`;
- `OR`;
- `NOT`.

Esempio:

```sql
SELECT *
FROM ordini
WHERE costo > 100 AND prodotto = 'forno';
```

> 📌 `WHERE` controlla quali righe compaiono nel risultato.

### **8. Proiezione**

Una **proiezione** seleziona solo alcune colonne di una tabella.

Per esempio:

```sql
SELECT cognome, nome, citta
FROM clienti;
```

Questa query:
- legge la tabella `clienti`;
- non pone condizioni sulle righe;
- restituisce solo tre attributi;
- mantiene lo stesso numero di righe della tabella di partenza, salvo duplicati e dettagli legati al modello SQL.

Il risultato è una tabella con meno colonne rispetto a `clienti`.

<!-- INSERT INSTRUCTOR SLIDE/DIAGRAM HERE -->

> 📌 La proiezione riduce le colonne, non necessariamente le righe.

---

### **9. Restrizione o Selezione delle Righe**

Una restrizione seleziona solo le righe che soddisfano una condizione.

Per esempio:

```sql
SELECT *
FROM ordini
WHERE costo > 100;
```

Questa query:
- legge la tabella `ordini`;
- restituisce tutte le colonne, perché usa `*`;
- restituisce solo le righe che soddisfano la condizione.

Se una sola riga soddisfa la condizione, il risultato contiene una sola riga.

Se nessuna riga soddisfa la condizione, il risultato è una tabella vuota.

<!-- INSERT INSTRUCTOR SLIDE/DIAGRAM HERE -->

> 📌 La restrizione riduce le righe, non necessariamente le colonne.

---

### **10. Combinare Proiezione e Restrizione**

Una query può combinare proiezione e restrizione.

Esempio:

```sql
SELECT prodotto, costo
FROM ordini
WHERE prodotto = 'forno';
```

Questa query:
- legge la tabella `ordini`;
- restituisce solo gli attributi `prodotto` e `costo`;
- restituisce solo le righe in cui il prodotto è `forno`.

Il risultato è quindi filtrato sia sulle colonne sia sulle righe.

> 💡 Molte query reali sono proprio combinazioni di colonne richieste e condizioni sulle righe.

---

### **11. Restituire l'Intera Tabella**

Se si usa `*` e si specifica una sola tabella nel `FROM`, senza `WHERE`, la query restituisce l'intera tabella.

Esempio:

```sql
SELECT *
FROM clienti;
```

Il risultato contiene:
- tutte le colonne;
- tutte le righe.

Questa forma è comoda per esplorazione e debugging, ma nelle applicazioni reali può essere inefficiente o rischiosa se la tabella contiene molti dati o dati sensibili.

> ⚠️ In un'applicazione web, restituire più dati del necessario può aumentare l'esposizione in caso di errore o vulnerabilità.

---

### **12. Query su Più Tabelle**

La clausola `FROM` può contenere più tabelle.

Quando si combinano più tabelle, si ottiene concettualmente un prodotto tra tabelle, poi filtrato dalle condizioni.

Esempio:

```sql
SELECT clienti.nome, ordini.prodotto
FROM clienti, ordini
WHERE clienti.codiceCliente = ordini.codiceCliente;
```

Questa query:
- usa la tabella `clienti`;
- usa la tabella `ordini`;
- restituisce il nome del cliente e il prodotto ordinato;
- mantiene solo le righe in cui il codice cliente coincide.

La condizione nel `WHERE` serve a collegare correttamente i record delle due tabelle.

Senza una condizione adeguata, si otterrebbero combinazioni non significative.

> 📌 Quando si usano più tabelle, le condizioni di collegamento sono fondamentali per ottenere risultati significativi.

### **13. Struttura Generale del Comando SELECT**

Riassumendo, la struttura base del comando SQL per interrogare un database è:

```sql
SELECT lista_attributi
FROM lista_tabelle
WHERE lista_condizioni;
```

La `lista_attributi` indica le colonne da restituire.

La `lista_tabelle` indica da quali tabelle leggere.

La `lista_condizioni` indica quali proprietà devono soddisfare le righe.

La clausola `WHERE` può essere assente:

```sql
SELECT lista_attributi
FROM lista_tabelle;
```

---

### **14. Tabelle SQL come Multinsiemi**

Nel linguaggio SQL, una tabella non è necessariamente un insieme matematico.

È più correttamente un **multinsieme**.

Questo significa che possono esserci righe uguali.

Se si vuole eliminare duplicati dal risultato, si può usare:

```sql
SELECT DISTINCT attributi
FROM tabelle;
```

`DISTINCT` indica che si vogliono valori distinti.

> 📌 SQL può restituire duplicati: `DISTINCT` serve quando si vuole un risultato senza righe duplicate.

---

### **15. Qualificare gli Attributi**

Quando si lavora con più tabelle, possono esserci attributi con lo stesso nome.

Per evitare ambiguità bisogna qualificare gli attributi.

Se una tabella si chiama `R` e contiene un attributo `A`, si può scrivere:

```sql
R.A
```

cioè:

```text
nome_tabella.nome_attributo
```

È anche possibile usare un identificativo o alias della tabella.

Esempio:

```sql
SELECT C.nome, O.prodotto
FROM clienti C, ordini O
WHERE C.codiceCliente = O.codiceCliente;
```

Qui:
- `C` è un alias per `clienti`;
- `O` è un alias per `ordini`;
- `C.nome` indica l'attributo `nome` della tabella `clienti`;
- `O.prodotto` indica l'attributo `prodotto` della tabella `ordini`.

> ⚠️ Qualificare gli attributi è importante quando tabelle diverse hanno colonne con lo stesso nome.

---

### **16. Terminazione dei Comandi**

Come in molti linguaggi, i comandi SQL vengono separati usando il punto e virgola:

```sql
SELECT *
FROM clienti;
```

Il punto e virgola indica la fine del comando.

Dal punto di vista della SQL injection, questo dettaglio è importante perché un attaccante può tentare di chiudere una query e aggiungere un secondo comando.

> 📌 Anche la sintassi di separazione dei comandi è rilevante per capire come un input malevolo può alterare una query.

---

### **17. SQL non è Solo Interrogazione**

Nonostante il nome **Structured Query Language**, SQL non serve solo a interrogare una base di dati.

SQL è anche usato per:
- creare basi di dati;
- creare tabelle;
- inserire dati;
- modificare dati;
- eliminare dati;
- modificare la struttura del database.

In termini generali, SQL include componenti di:
- **Data Definition Language**;
- **Data Manipulation Language**.

Questo è centrale per la sicurezza.

Se un attaccante riesce a far eseguire SQL arbitrario, non può solo leggere dati: può anche modificarli o eliminarli.

---

### **18. Comando DROP**

Il comando `DROP` serve a eliminare oggetti dal database.

Per eliminare una tabella:

```sql
DROP TABLE nome_tabella;
```

Per eliminare un intero database:

```sql
DROP DATABASE nome_database;
```

Questi comandi sono estremamente distruttivi.

In un attacco di SQL injection, riuscire a iniettare un comando `DROP` può portare alla perdita di dati o alla compromissione completa dell'applicazione.

> ⚠️ `DROP` è uno dei motivi per cui SQL injection è pericolosa: l'attaccante potrebbe non limitarsi a leggere, ma distruggere la base dati.

---

### **19. Comando INSERT**

Il comando `INSERT` permette di inserire nuove righe in una tabella.

Una forma tipica è:

```sql
INSERT INTO nome_tabella (attributo1, attributo2, attributo3)
VALUES (valore1, valore2, valore3);
```

Esempio concettuale:

```sql
INSERT INTO clienti (cognome, nome, citta)
VALUES ('Rossi', 'Mario', 'Milano');
```

`INSERT` è rilevante per la sicurezza perché un attaccante potrebbe tentare di:
- aggiungere utenti;
- inserire dati falsi;
- creare account amministrativi;
- manipolare record applicativi;
- inquinare log o tabelle.

---

### **20. Collegamento con SQL Injection**

La prossima lezione tratterà gli attacchi di tipo SQL injection.

Il problema nasce quando un'applicazione costruisce query concatenando stringhe e input utente.

Esempio concettuale:

```sql
SELECT *
FROM utenti
WHERE username = 'input_utente';
```

Se `input_utente` viene inserito nella query senza controllo, un attaccante può fornire un valore costruito per cambiare il significato della query.

Per capire questi attacchi serve conoscere:
- struttura `SELECT-FROM-WHERE`;
- uso degli operatori booleani;
- effetto di `OR`;
- uso del punto e virgola;
- presenza di comandi distruttivi come `DROP`;
- differenza tra dati e codice SQL.

> 📌 SQL injection consiste nel far interpretare come codice SQL ciò che l'applicazione avrebbe dovuto trattare come semplice dato.

---

### **21. Riepilogo**

> ✅ SQL permette di interrogare e gestire basi di dati relazionali. Una base di dati relazionale è composta da tabelle; le righe sono record e le colonne sono attributi. Una query SQL restituisce una tabella, anche vuota. La forma base di interrogazione usa `SELECT` per scegliere le colonne, `FROM` per scegliere le tabelle e `WHERE` per specificare condizioni sulle righe. SQL lavora su multinsiemi, quindi può produrre duplicati salvo uso di `DISTINCT`. Quando si combinano più tabelle bisogna qualificare gli attributi con il nome della tabella o un alias. SQL non serve solo a interrogare: permette anche di modificare e definire dati e strutture, tramite comandi come `INSERT` e `DROP`. Questa potenza rende SQL injection particolarmente pericolosa: un input malevolo può cambiare il significato di una query e, se l'applicazione è vulnerabile, leggere, modificare o distruggere dati.
